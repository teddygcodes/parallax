# PARALLAX — Full Build Prompt for Claude Code

---

## What You Are Building

**PARALLAX** is a real-time global conflict intelligence platform.

Tagline: *"Where reality and narrative diverge."*

It shows three wars simultaneously:
1. **The Physical War** — missiles, drones, ships, troops on a live 3D globe
2. **The Information War** — the same event reported differently across Western, Russian, Middle Eastern, and OSINT sources
3. **The Truth War** — how narratives evolve, shift, and converge over time

This is not a dashboard. It is not another news aggregator. It is a cinematic, meditative experience that makes you feel like you are watching the world from space — and slowly realizing what you're actually seeing.

---

## Design Direction — READ THIS FIRST

**The aesthetic is: Cold War situation room meets planetarium.**

Do not use default Tailwind styling. Every design decision should feel like it was made by a human designer with a specific vision, not generated by AI.

### Tone
Deliberate. Cold. Cinematic. The restraint is the point. This feels like the opening sequence of a Fincher film crossed with a JPL mission control screen. Beautiful because the data itself is the design, not decoration around it.

### Typography
- **Headers / Globe labels:** `Bebas Neue` or `Editorial New` — weight and presence
- **Data readouts / timestamps / coordinates:** `IBM Plex Mono` — everything feels like telemetry
- **Body / analysis text:** `Instrument Serif` or `Playfair Display` — editorial authority
- **Never:** Inter, Roboto, Arial, system fonts, Space Grotesk

### Color Palette
```
--bg-void:          #0a0a0e   /* base background — not pure black */
--globe-ocean:      #0d1b2a   /* deep navy */
--globe-land:       #141c26   /* barely lighter than ocean */
--globe-atmosphere: #1a3a5c   /* soft rim glow */
--event-strike:     #c0392b   /* muted crimson — airstrikes, missiles */
--event-naval:      #d4a017   /* amber — ships, fleet */
--event-drone:      #8e44ad   /* dim violet — drone activity */
--event-cyber:      #00b894   /* cold teal — cyber events */

--disputed:         #e17055   /* burnt orange */
```

### Motion Principles
- Everything is slow. Nothing snaps or pops.
- Globe rotates at 0.05 degrees per frame — barely perceptible
- Events appear with a soft bloom pulse, expand, then fade over ~8 seconds
- Camera drift in Living Earth Mode: slow, damped, feels observed not controlled
- Hover states: subtle brightness shift, no dramatic transforms
- Page transitions: fade only, no slides

### Layout Principles
- Massive negative space — let the globe breathe
- Minimal chrome — no heavy nav bars, no sidebars cluttering the globe
- Data cards appear as overlays, not panels
- Everything labeled like instrument readouts:
```
31.343°N  34.305°E  ·  18:03 UTC  ·  STRIKE CONFIRMED
```

## Tech Stack

**Frontend:**
- React + Next.js (App Router)
- `globe.gl` — 3D globe base layer (handles arcs, pulses, atmosphere glow, camera — do NOT build from raw Three.js)
- `deck.gl` — geospatial data layers on top of globe (tension heatmap, event density, clustering visualization)
- Framer Motion — UI panel transitions and animations
- Tailwind CSS — layout only, all visual styling via CSS variables and custom classes

**Why globe.gl over raw Three.js:**
Globe.gl already solves arc math, lat/lng projection, atmosphere rendering, animated pulses, and camera controls. Building these from scratch in raw Three.js wastes weeks on solved problems. Use globe.gl as the base, extend with deck.gl layers for density and heatmap effects.

```javascript
// Globe.gl gets missile arcs working in an afternoon
import Globe from 'globe.gl';

const globe = Globe()(document.getElementById('globe'))
  .globeImageUrl('dark-earth-texture.jpg')
  .backgroundColor('#0a0a0e')
  .arcsData(missileEvents)
  .arcColor(() => '#c0392b')
  .arcDashLength(0.5)
  .arcDashAnimateTime(4000)
  .pointsData(strikeEvents)
  .pointColor(() => '#c0392b')
  .pointAltitude(0.01);
```

**Backend:**
- FastAPI (Python)
- PostgreSQL — event storage, narrative history, claim tracking
- Redis — real-time event caching
- Celery — background data ingestion workers

**AI:**
- Anthropic API (`claude-sonnet-4-20250514`) — neutral analysis, claim extraction, divergence scoring, coordinated messaging detection

**Data Feeds (v1 — start here, expand later):**
- ACLED API — real-time conflict events (strikes, missiles, drones)
- NewsAPI / GDELT — global news aggregation
- RSS — state media (RT, CGTN, IRNA, TASS, Al Jazeera, BBC, Reuters, AP)

**Data Feeds (v2 — add after v1 is stable):**
- MarineTraffic API — naval positions
- ADS-B / FlightAware — air activity

## Data Architecture — Build This First

**Event identity and clustering is the foundation. Nothing else works without it.**

Every feature (Narrative Replay, Divergence Meter, Claim Tracking) breaks if two reports about the same strike are treated as two separate events.

### Data Model

```
EVENT
  id                    EVT-2026-000384
  coordinates           [lat, lng]
  first_detection_time  UTC timestamp
  event_type            STRIKE | MISSILE | DRONE | NAVAL | TROOP
  confidence            VERIFIED | LIKELY | REPORTED | UNCONFIRMED | DISPUTED
  cluster_radius        km

SIGNALS (raw incoming reports that feed events)
  id
  event_id              → EVENT
  source                Reuters | RT | IRNA | Bellingcat | etc.
  source_category       WESTERN | RUSSIAN | MIDDLE_EAST | OSINT | LOCAL
  article_url
  published_at          UTC timestamp
  raw_text
  coordinates_mentioned
  description

CLAIMS (specific assertions extracted from signals)
  id
  event_id              → EVENT
  claim_text            "Strike targeted military depot"
  source                who made this claim
  source_category
  first_seen_at
  status                VERIFIED | UNVERIFIED | DISPUTED | DISPROVEN | COORDINATED_MESSAGING_SUSPECTED
  confidence_score      0.0–1.0

NARRATIVE_HISTORY (how claims evolve over time)
  id
  claim_id              → CLAIM
  timestamp
  status_before
  status_after
  trigger               what caused the update (satellite imagery / OSINT / official statement)
  notes
```

### Event Clustering Logic

Incoming signals cluster into events using:
- Distance threshold: < 5km
- Time window: < 60 minutes
- Event type similarity scoring

```python
# Pseudo-logic
if signal.distance_to_nearest_event < 5km
   and signal.time_delta < 60min
   and signal.type_similarity > 0.7:
     merge_into_existing_event()
else:
     create_new_event()
```

Human override available for edge cases.

## The Two Tabs

---

### TAB 1 — THE GLOBE

**The hook. The wow moment. The thing that makes someone stop scrolling.**

A 3D sphere filling most of the viewport. Deep void background. Soft atmospheric rim glow. Landmasses barely distinguishable from oceans — almost monochromatic until events appear.

**Event visual language:**
| Event Type | Visual |
|---|---|
| Missile launch | Clean arc traced across atmosphere with trailing glow |
| Airstrike | Expanding ring pulse, blooms and fades over 8s |
| Drone activity | Quick flicker, dim violet |
| Naval position | Slow-moving amber dot constellation |
| Troop movement | Soft flowing line, barely visible |

**Interaction:**
- Hover event → minimal telemetry card appears
```
STRIKE  ·  CONFIRMED
31.343°N  34.305°E
18:03 UTC  ·  Mar 4 2026
Source: ACLED + Bellingcat

Signals received:    37
Unique sources:      12
Geographic spread:   4 regions
```
Higher signal density = more documented event. Low signal density = watch carefully, story may be thin.
- Click event → opens Narrative Panel (slides in from right, globe dims slightly)
- No click needed to enjoy — it works as pure ambient visualization

**Tension Glow:**
Regions with sustained activity over the last 24hrs develop a soft regional glow — brighter = more events. No labels needed. Tension is visible.

**Controls (minimal, bottom of screen):**
- Time Rewind slider — drag back 12h / 24h / 72h / 1 week
- Event type toggles — show/hide strikes, naval, drones, etc.
- Living Earth Mode toggle

---

### LIVING EARTH MODE

Passive ambient mode. No interaction needed.

- All UI chrome disappears
- Globe rotates slowly
- Events appear and fade in silence
- Camera drifts to new regions every 90 seconds — damped, observatory-like
- One soft low tone plays when a significant new event appears
- Time Rewind available: "Last 72 hours of global conflict" as a loop
- **"Record 30s" button** — captures canvas via MediaRecorder API and exports as MP4. This is the viral mechanic. A 30-second clip of the globe with arcs and pulses posted on X is the entire marketing strategy.

**This is what people record and post.** Build the record button earlier than you think you need it.

### FULL ANALYSIS VIEW

**1. Divergence Meter**
```
Consensus ●━━━━━━━━━━━━━━━━━━━━━━━━● Contested
          [████████████░░░░░░░░░░░░░]  62%
```
Color shifts green → amber → crimson. Label below: LOW / MODERATE / HIGH / EXTREME divergence.

**2. Side By Side Sources**
Four columns:
- 🇺🇸 Western (Reuters, AP, BBC, NYT)
- 🇷🇺 Russian / Chinese (RT, TASS, CGTN)
- 🌍 Middle East (Al Jazeera, IRNA, Press TV)
- 🔍 OSINT / Independent (Bellingcat, local)

Each column shows headline + 2-sentence summary + confidence badge. No editorial from the platform.

**3. Information Confidence Tags**
```
✅ Verified              — confirmed by multiple independent sources + imagery
🔵 Likely               — corroborated but not fully confirmed
🟡 Reported             — single source, plausible
⚪ Unconfirmed          — early report, not corroborated
🔴 Disputed             — sources actively contradict each other
⚠️ Coordinated Messaging Suspected — AI flagged patterns consistent with state-coordinated messaging
```

Note: Do not use the word "Propaganda" anywhere in the UI. It is politically charged and will undermine trust in the platform's neutrality. "Coordinated Messaging Suspected" is accurate, defensible, and less absolute.

**4. AI Analysis Panel**
Neutral. Clinical. No opinion. No political leaning.

Format:
```
WHAT IS CONFIRMED
Explosion detected at coordinates [x,y] at 18:03 UTC. 
Geolocated video verified by OSINT sources.

WHAT IS DISPUTED  
Target type: Western outlets report military depot. 
Russian outlets report civilian building.
Casualty figures: range from 0 (official) to 40+ (local sources).

WHERE INFORMATION GOES DARK
No independent access to site. Satellite imagery expected within 6-12 hours.

CORE DISAGREEMENT
Who launched the strike and what was the intended target.
```

**5. Claim Tracker**
Individual claims extracted from all sources, tracked over time:

```
CLAIM: "Strike targeted a weapons depot"
Origin: Russian MoD statement  ·  18:45 UTC
Status: DISPUTED
Outlets repeating: 14
Outlets contradicting: 9
Last updated: 2h ago
```

## Build Order

Do not start with the globe. Start with the data.

**Phase 1 — Data Foundation**
1. Project structure, environment, dependencies
2. PostgreSQL schema — EVENT, SIGNALS, CLAIMS, NARRATIVE_HISTORY tables
3. ACLED API ingestion — pull real conflict events into DB
4. Event clustering logic — deduplicate and merge signals into events
5. NewsAPI / RSS ingestion — pull articles per event
6. Anthropic API integration — claim extraction, divergence scoring, AI analysis

**Phase 2 — The Globe**
7. Globe.gl setup — sphere, atmosphere glow, basic rotation, color palette applied
8. Plot events on globe — arcs (missiles), pulses (strikes), flickers (drones)
9. Deck.gl tension layer — regional heatmap based on event density
10. Hover cards — telemetry-style overlays with Signal Density
11. Time Rewind slider
12. Living Earth Mode — camera drift, ambient sound, minimal chrome
13. MediaRecorder "Record 30s" button in Living Earth Mode

**Phase 3 — The Signal + Share**
14. Headline feed UI — minimal list, divergence dot indicators
15. Full analysis view — side by side sources, divergence meter component
16. Confidence tagging system (no "Propaganda" label — use "Coordinated Messaging Suspected")
17. AI Analysis Panel — Anthropic API call, formatted output
18. Claim Tracker component
19. Narrative Replay timeline — scrubber, event history rendering
20. Share Cards — Region Snapshot, Single Event, Divergence cards with export

**Phase 4 — Polish**
21. Typography refinement — Bebas Neue, IBM Plex Mono, Instrument Serif locked in
22. Motion audit — all animations reviewed for restraint, nothing under 300ms
23. Sound design — ambient tones, event sounds, silence respected
24. Mobile optimization — stacked columns, simplified globe controls
25. Performance — Redis caching, lazy-load globe textures, cap draw calls

## Design Rules (Non-Negotiable)

1. If it adds clutter, it doesn't ship
2. No pure white (`#ffffff`) or pure black (`#000000`) anywhere
3. No Inter, Roboto, Arial, or system fonts
4. All data displayed in monospace — coordinates, timestamps, counts
5. All animations are slow — nothing under 300ms, most things 600ms–1200ms
6. The globe is always visible. Never fully covered by UI panels.
7. Silence between events is intentional. Do not fill it.
8. Every element earns its place. Default to removing, not adding.
